## Stack

Full-stack shopping list app: SvelteKit 2 + Svelte 5 + Tailwind v4 frontend, Kotlin + Spring Boot 4 + Spring Data JPA + Flyway backend, PostgreSQL 17. Java 25 required.

## Commands

Backend (`backend/`):
- `./gradlew bootRun` тАФ run app on :8080. Auto-starts Postgres via Spring Boot's Docker Compose integration (uses `compose.yaml`); no manual `docker compose up` needed.
- `./gradlew test` тАФ run tests. Single test: `./gradlew test --tests "com.grabit.lists.ShoppingListServiceTest"`.
- `./gradlew build` тАФ full build incl. tests.

Frontend (`frontend/`):
- `npm run dev` тАФ Vite dev server on :5173.
- `npm run check` тАФ type-check with svelte-check (no separate lint script).
- `npm run build` тАФ production build.

## Architecture

### Backend layering (`backend/src/main/kotlin/com/grabit/`)
- `lists/` is the single feature module: Entity тЖТ Repository тЖТ Service тЖТ Controller, with DTOs in `dto/` and entityтЖФDTO extension functions in `Mappers.kt`. **DTOs only cross the controller boundary** тАФ entities are never serialized.
- JPA entities are plain `class` with mutable fields (NOT `data class`) to avoid `equals/hashCode` recursion across the bidirectional listтЖФitems relationship.
- `spring.jpa.open-in-view: false` plus `@EntityGraph(attributePaths = ["items"])` on read repos prevents N+1.
- Flyway owns the schema (`src/main/resources/db/migration/V*.sql`); Hibernate runs `ddl-auto: validate`. Schema changes = new `V{n}__*.sql` migration, not entity-driven DDL.
- UUID v4 PKs generated by Hibernate 7 native.
- `common/GlobalExceptionHandler.kt` shapes all error responses into the `ApiError` JSON format.

### Frontend data flow (`frontend/src/`)
- **Local-first**: `+layout.ts` (with `ssr = false`) loads all lists once at app start and hydrates the Svelte store. Routes thereafter read from the store тАФ navigation is instant, not network-bound.
- **Optimistic mutations with revert-on-failure**: every mutation in `lib/stores/shopping.svelte.ts` updates state immediately, fires the API call via `lib/api/`, then reconciles with the server response or reverts + toasts on failure. When adding new mutations, follow this pattern тАФ don't await before updating UI.
- **Sync indicator** (`lib/stores/sync.svelte.ts`) uses a counter-based `inFlight` so "SavingтАж" stays stable through bursts of overlapping requests.
- Svelte 5 class stores with `$state` / `$derived` runes тАФ not the legacy `writable()` API.
- Tailwind v4 via `@tailwindcss/vite`; theme tokens are in `src/app.css` under `@theme` тАФ there is no `tailwind.config.js`.

### Cross-cutting
- CORS is configured in `backend/.../config/CorsConfig.kt` for `http://localhost:5173`.
- API base URL on the frontend is `PUBLIC_API_BASE_URL` (default `http://localhost:8080`).
- See README.md for the full HTTP API table and error envelope.
